def read_cars_data():
    path = r'./Data/impressions/' 
    all_files = glob.glob(path + "/*.csv")
    li = []
    for filename in all_files:
        df = pd.read_csv(filename, index_col=None, header=0, sep=',', low_memory=False)
        li.append(df)
    cars_data = pd.concat(li, axis=0, ignore_index=True)
    cars_data.CAR_OWNER_PRR.replace({"\\\\N": 0.5}, inplace=True)
    return cars_data

def generate_reduced_data(request_id, data):
    return data[data.SEARCH_REQUEST_ID == request_id]

def get_distance_score(data, car_id):
    #distance score:
    column = 'CAR_DISTANCE_TO_SEARCH_ADDRESS'
    max_value = data.loc[data[column].idxmax()][column]
    min_value = data.loc[data[column].idxmin()][column]
    x_value = data.loc[data['CAR_ID'] == car_id][column].tolist()[0]
    factor_2 = max_value - x_value
    if factor_2 > 0:
        return score_base / ((max_value - min_value) / factor_2) + epsilon
    else:
        return epsilon

def get_owner_prr(cars_data, car_id):
    column = 'CAR_OWNER_PRR'
    return float(cars_data.loc[cars_data['CAR_ID'] == car_id][column].tolist()[0]) + epsilon
    
def get_price_score(cars_data, car_id):
    column = 'CAR_SEARCH_PRICE'
    max_value = cars_data.loc[cars_data[column].idxmax()][column]
    min_value = cars_data.loc[cars_data[column].idxmin()][column]
    x_value = cars_data.loc[cars_data['CAR_ID'] == car_id][column].tolist()[0]
    factor_2 = max_value - x_value
    if factor_2 > 0:
        return score_base / ((max_value - min_value) / factor_2) + epsilon
    else:
        return epsilon
    
    
def get_price_bonus_score(requests_data, cars_data, car_id):
    try:
        usr_avg_price = requests_data['USER_AVG_BOOKING_PRICE'].tolist()[0]
        usr_avg_duration = requests_data['USER_AVG_BOOKING_DURATION'].tolist()[0]
        if (type(usr_avg_price) is str or not math.isnan(usr_avg_price)) and (type(usr_avg_duration) is str or not math.isnan(usr_avg_duration)):
            column = 'CAR_SEARCH_PRICE'
            usr_avg_price = float(usr_avg_price)
            usr_avg_duration = float(usr_avg_duration)
            current_duration = cars_data.loc[cars_data['CAR_ID'] == car_id][column].tolist()[0]
            estimated_price = (usr_avg_price/usr_avg_duration) * (requests_data['SEARCH_DURATION_IN_DAYS'].tolist()[0])
            #print("estimated_price ",estimated_price)
            min_proposed_price = cars_data.loc[cars_data[column].idxmin()][column]
            #print("min ",min_proposed_price)
            max_proposed_price = cars_data.loc[cars_data[column].idxmax()][column]

            if current_car_price > estimated_price:
                return 0

def get_user_preferences(search_request_id, data):
    try:
        result = dict()
        nb_all_bookings = int(data['USER_NB_BOOKINGS'].tolist()[0])
        nb_open_bookings = int(data['USER_NB_OPEN_BOOKINGS'].tolist()[0])    
        nb_instant_bookings = int(data['USER_NB_INSTANT_BOOKINGS'].tolist()[0]) 
        result['OPEN_TRUE'] = nb_open_bookings / nb_all_bookings
        result['OPEN_FALSE'] = 1 - result['OPEN_TRUE']
        result['INST_TRUE'] = nb_instant_bookings / nb_all_bookings
        result['INST_FALSE'] = 1 - result['INST_TRUE']
        return result
    except:
        return None

Explaining the scoring system: From analysing the data, we notice a feature which represents the current ranking system based on current score. The task being not to replicate the current ranking, this system cannot be used as a ground-truth model. Any supervied models cannot be applied on this given column.

The process i used in order to create my scoring model is by following these steps:

  • Ask colleagues and friends to rank based on the importance of factors in renting a car, these being : price, distance, The car model, car age
  • Try Drivy App in real life situations in order to analyse the current behaviour.
  • Check online (Kaggle, Quora mostly) for similar Problems/Models and analyse the proposed approach; a similar issue being 'Renting an apartement for holidays'

My proposed approch works as follow:

Let $ S(car\_id) $ be a function that return a probablibity score between 0 and 1. Which is definied as follow :

$$ S(car\_id) = ( \frac{1}{n} \sum_{\substack{ 0<i<n }} w_i * P(x_i) ) * R(car\_id) $$

For each feature $x_i \in \{car\_price, car\_distance, car\_age, user\_preference_open, user\_preference_inst \} $

Where :

$ w_i $ Weight of $x_i$ that represents its importance

$ R(car\_id) $ : the car's owner acceptance rate

Let's dive deeper for each $x_i$ :

1- Car price : The probability of a car being selected based on price only is calculated based on three parameters:

  • $ P_{car\_price_1}(car\_id) $ The car's price based on the request: with $0+ \epsilon$ being the probability of choosing the car with the highest price, $1$ for chosing the car with the lowest price

  • $ P_{car\_price_2}(car\_id) $ The car's extra price : with $0+ \epsilon$ being the probability of choosing the car with the highest price, $1$ for chosing the car with the lowest price

  • $ P_{car\_price_3}(car\_id) $ The current offer compared to the usual renting price by the user, with $0+ \epsilon$ attributed to the offers that are either higher or equal to the usual renting price, $1$ for the car with the best offer compared to the usual price
    • The score is is calculated as follow: $$ P(x_i = car\_price) = \frac{1}{3} \sum_{\substack{ 0<i<=3 }} P_{car\_price_i}(car\_id) $$

    2- Car distance / Car age: The probability of a car being selected based on distance only or age only is calculated as follow: $$ P(x_i = car\_distance) = \begin{cases} 0+ \epsilon & \quad \text{Furthest / oldest car in the pool}\\ 1 & \quad \text{Closest / newest car in the pool} \end{cases} $$

    3- User preferences: This attribute is calculated as follow:

  • Probability of the the car being selected based on its OPEN value, which is calculated as following
    • $$ P(x_i = user\_pref\_open) = \begin{cases} P(x_i / V(car\_opened)=true ) & \quad \text{If V(car_opened)=true : return pourcentage of opened car booked by the user}\\ 1-P(x_i / V(car\_opened)=true ) & \quad \text{If V(car_opened)=false} \end{cases} $$

  • Probability of the the car being selected based on its INSTANT_BOOKABLE value, which is calculated as following
    • $$ P(x_i = user\_pref\_inst) = \begin{cases} P(x_i / V(car\_inst)=true ) & \quad \text{If V(car_inst)=true : return pourcentage of intant bookable cars booked by the user}\\ 1-P(x_i / V(car\_inst)=true ) & \quad \text{If V(car_inst)=false} \end{cases} $$

    Critics and future ideas

    Critics¶

    The challenge with working on this particular scoring model, is that there isn't a ground truth model upon which we can evaluate our results. The scoring model proposed has some flaws which are:

  • Considering that features are independant. For example, based on the correlation graph, $cov(car\_is\_open, car\_instant\_bookable) = 0.870173 $ which means that these features are highly corelated. Correlation and causality should be checked before calculating the probabilty.
  • Weights $w_i$ are chosen based on a small survey, these value can bias the scoring model and should be reconsidered

    • Future ideas¶

  • Create user profiles where each profile is defined by a set of renting habbits, criterias importance (prince, distance...)
  • Capture what the user is really looking for, based on his history or/and using the behaviour of it profiling data
  • Create a score of each car model, to try to propose cars that are highly demanded
  • Utilize cars ratings, and use NLP methods on user's feedback to improve the rating
  • Create a score on car owners and prioritize well rated owners
